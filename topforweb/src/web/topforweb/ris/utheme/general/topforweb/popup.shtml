Codice HTML

Gli elementi XHTML che permettono di aprire e chiudere il blocco faq sono descritti qui sotto:

<div class="faq"> : definisce il blocco della singola faq.
<div class="risposta"> : definisce il contenitore della risposta nascosta del blocco faq.

        <div class="faq">                                       
        	<h4>Domanda faq 1</h4>                                            
        	<div class="risposta">                                              
        		<p>Risposta faq 1</p>
        	</div>                                        
        </div> 
                                                                             
        <div class="faq">                                            
        	<h4>Domanda faq 2</h4>                                            
        	<div class="risposta">                                               
        		<p>Risposta faq 2</p>
        	</div>                                                                  
        </div>
La Pagina

Questa pagina delle Faq presenta già gli stili XHTML per l'eventuale apertura in una dialog (vedere snippet "Dialog", paragrafo "pagina richiamata").

La pagina copiata dovrà essere posizionata nella cartella delle risorse statiche (dove ci sono le pagine help e contatti), apportando le opportune modifiche per collegarla al proprio servizio (vedere Per ogni pagina degli snippets copiata ricordarsi di):
/ris/resources/application/nome_servizio/statiche/faq.shtml

Fogli di stile

Includere i css dei pluging nell'head.html del servizio dopo l'ultimo della lista dei css caricati via js (vedere scheda Caricamento css da jquery tramite tag media):

        <!-- caricamento css per JQUERY per le faq--> 
        <link href="/ris/resources/global/css/jquery/faq.css" rel="stylesheet" type="text/css" media="javascript-screen" />

Nel caso di Sistema Piemonte alcune definizioni sono sovrascritte in un altro css.
Si CREA il css specifico del servizio /ris/resources/application/nome_servizio/css/jquery/z_jquery_all_SP.css e lo si collega nell'head.html del servizio come ultimo nell'elenco dei css.

        <!--SOVRASCRITTURE SP-->		
		<link href="/ris/resources/application/patterns/css/jquery/z_jquery_all_SP.css" rel="stylesheet" type="text/css" media="javascript-screen" /> 
			
        <!-- ////////// FINE CSS -->

Dentro il foglio di stile z_jquery_all_SP.css si collega il foglio di stile che va a sovrascrivere quello di Rupar (il nome è uguale ed ha in più il suffisso _SP): 

       @import url(/ris/resources/global/css/jquery/faq_SP.css);

Cosa sono le credenziali d'accesso?

Le credenziali sono il nome utente, la password, il PIN e il certificato digitale. Potete autenticarvi a partire da qualsiasi pagina del sito: è sufficiente cliccare sul link ENTRA (presente nella testata di ogni pagina del portale in alto a destra)  e poi fornire le proprie credenziali d'accesso.

Come si ottengono le credenziali per accedere?

Per richiedere le credenziali d'accesso è necessario compilare il modulo ed inviarlo via e-mail all'indirizzo prova@csi.it
Modulo per la richiesta delle credenziali

Il processo di autenticazione è sicuro?

Quando vi autenticate trasmettete a RUPAR i vostri dati personali, che non devono essere "letti" da nessun altro.
Per questo la trasmissione avviene utilizzando tecnologie avanzate (crittografia SSL-128 bit e infrastruttura a chiave pubblica) in grado di criptare i dati in transito sulla rete, attraverso l'uso di Certificati Digitali.
Questa tecnologia, tra le migliori tecniche di sicurezza per le transazioni, viene utilizzata da molte banche per l'erogazione dei servizi telematici ai propri clienti ed è indicata dal Governo come strumento di riferimento per la Firma Digitale (Dpr. 513/97) e per la Carta di Identità Elettronica (Dpr. 437/99).
